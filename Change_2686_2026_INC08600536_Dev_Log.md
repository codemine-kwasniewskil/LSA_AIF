## 5a6cf93 — fix: inline DATA() in IMPORTING clause not allowed

### Problem
`create_csv_err_body` declared variables inline inside the `IMPORTING` clause of a functional method call:

```abap
DATA(lv_status) = get_processing_status(
  IMPORTING
    et_msgs      = DATA(lt_msgs)     " ← syntax error
    ev_kassz     = DATA(lv_kassz)
    ev_sap_objid = DATA(lv_sap_objid)
    ev_netdt     = DATA(lv_netdt) ).
```

The ABAP compiler rejects `DATA(var)` inside an `IMPORTING` clause of a functional method call in this compiler version.

### Fix
Moved `lt_msgs`, `lv_kassz`, `lv_sap_objid`, `lv_netdt` to an explicit `DATA:` block at the top of the method — the same pattern used in the adjacent `create_lst_body` method.

## ae46a3b — fix: lv_errtxt TYPE string incompatible with escape_csv iv_value

### Problem
`lv_errtxt` was declared inline with a backtick literal:

```abap
DATA(lv_errtxt) = ``.
```

The `MESSAGE ... INTO lv_errtxt` statement requires a character-type target in this compiler version. This caused type ambiguity: the inline declaration produced a type that was incompatible with `escape_csv( iv_value TYPE STRING )`.

### Fix
Added `lv_errtxt TYPE string` to the explicit `DATA:` block at method top. Replaced the inline init with `CLEAR lv_errtxt.` inside the loop.

## d916f0a — fix: tighten INITIAL-status error condition in create_csv_err_body

### Problem
The original error filter:

```abap
CHECK lv_status = 'E' OR lv_status = 'A' OR lv_status IS INITIAL.
```

was too broad. `get_processing_status` returns `rv_status = INITIAL` in two distinct cases:

| Case | `lv_status` | `lt_msgs` | Should appear in CSV? |
|------|-------------|-----------|----------------------|
| AO found, FI posting failed silently | INITIAL | has E entries | **Yes** |
| No AO/MB/VR record found for GLBLID | INITIAL | empty | **No** |

Plain `IS INITIAL` included both, so records with no document at all were written as errors.

### Fix
Gate the INITIAL case on whether `lt_msgs` carries an E or A message:

```abap
CHECK lv_status = 'E' OR lv_status = 'A'
   OR ( lv_status IS INITIAL
        AND ( line_exists( lt_msgs[ type = 'E' ] )
              OR line_exists( lt_msgs[ type = 'A' ] ) ) ).
```

## 25fa3ba — refactor: drop get_processing_status, read is_data-ao directly

### Problem
`create_csv_err_body` called `get_processing_status` which was designed for the LST protocol — a different use case. It performed work unnecessary for a CSV error report:

- Booking-key validation via `/THKR/AIF_ZALLGE_CHK_BTYP_SUPP`
- Treasury reference (Kassenzeichen) resolution via `get_treasury_id`
- Net due date resolution via `get_netdt_from_bseg`

Also, `lt_msgs`, `lv_kassz`, `lv_sap_objid`, `lv_netdt` were all declared but only `lt_msgs` was actually used.

### Investigation
Research confirmed that SAP AIF provides no standard per-GLBLID error mechanism suitable for this use case:
- `OLD_MESSAGES TYPE /AIF/BAL_T_MSG` is declared in all action FM signatures but never used by any custom FM — it is a flat BAL message list with no GLBLID field.
- `LOG_HANDLE` (from `FILE_GET_GLOBALS`) gives the full run-level BAL log but also without per-GLBLID structure.
- `/THKR/T_AIF_OBJ` has per-GLBLID status but is only written by `ACT_GP`, `ACT_MB`, `ACT_MB_UP` — not by `ACT_AO` which is what STIEWI/PTRAVEL/SAMBA use.

The correct and only reliable per-GLBLID error source for AO-based interfaces is `is_data-ao[glblid=...]-ao_proc_status` and `is_data-ao[glblid=...]-msg`.

### Fix
Replaced the `get_processing_status` call with a direct `READ TABLE is_data-ao WITH KEY glblid`. Removed the four now-unused variables.

**Note:** This version only covered `is_data-ao` — see next commit for full coverage.

## 07bdefc — feat: add GET_ERR_FOR_GLBLID, cover all document types

### Problem
The previous refactor read only `is_data-ao`, dropping coverage of:
- `is_data-ao_reference` — Anordnung Referenz
- `is_data-mb` — Mittelbindung
- `is_data-mb_up` — Mittelbindung Änderung
- `is_data-vr` — Verrechnungsanordnung
- `is_data-storno` — Storno

Interfaces using MB, VR, or Storno documents would produce an empty CSV even for genuine errors.

### Fix
Added private method `GET_ERR_FOR_GLBLID` to the class definition and implementation. It walks the same sub-table priority order as `get_processing_status` (`ao → ao_reference → mb → mb_up → vr → storno`) but returns only `ev_status` and `et_msgs` — no booking-key validation, no treasury or net-date lookups.

`create_csv_err_body` now calls `GET_ERR_FOR_GLBLID` instead of the inline AO read.

Method signature:

```abap
methods GET_ERR_FOR_GLBLID
  importing
    !IS_DATA   type /THKR/S_AIF_SAP
    !IV_GLBLID type /THKR/AIF_GLBLID
  exporting
    !EV_STATUS type /AIF/PROC_STATUS
    !ET_MSGS   type BAPIRET2_TT .
```

## a5a67b0 — refactor: GET_ERR_FOR_GLBLID — table expressions + IS ASSIGNED

### Problem
The initial implementation of `GET_ERR_FOR_GLBLID` used `READ TABLE ... WITH KEY` / `sy-subrc = 0`, which is pre-7.4 style.

### Fix
Replaced every `READ TABLE` block with ABAP 7.5 table expression syntax:

```abap
" Before
READ TABLE is_data-ao ASSIGNING FIELD-SYMBOL(<ls_ao>)
  WITH KEY glblid = iv_glblid.
IF sy-subrc = 0.

" After
ASSIGN is_data-ao[ glblid = iv_glblid ] TO FIELD-SYMBOL(<ls_ao>).
IF <ls_ao> IS ASSIGNED.
```

`ASSIGN itab[ key = val ] TO FIELD-SYMBOL(<fs>)` assigns the field symbol directly to the table row when the line exists; leaves it unassigned otherwise. `IS ASSIGNED` is safer than `sy-subrc = 0` because it cannot be overwritten by an intervening statement.

Structure used early-RETURN after first matching sub-table — only one document type's errors were reported per GLBLID.

## cca7401 — refactor: GET_ERR_FOR_GLBLID — line_exists + accumulate all sub-tables

### Problem
Two issues with the `a5a67b0` implementation:

1. **`ASSIGN` used instead of `line_exists`** — `ASSIGN itab[ key = val ] TO FIELD-SYMBOL(<fs>)` is unnecessarily low-level when the data is accessed only once. The idiomatic ABAP 7.5 pattern for a conditional read is `line_exists( itab[ key = val ] )` followed by a table expression for the value.

2. **Early `RETURN` after first matching sub-table** — A GLBLID can legitimately have errors in both `ao` and `mb` (e.g. the Anordnung posted but the Mittelbindung failed). The first-match-wins approach silently dropped all errors after the first hit. Status was not merged; messages from subsequent sub-tables were never collected.

### Fix
Replaced all `ASSIGN ... IS ASSIGNED ... RETURN` blocks with `line_exists( ... ) → IF` blocks. Removed all `RETURN` statements. All six sub-tables (`ao → ao_reference → mb → mb_up → vr → storno`) are always checked regardless of earlier hits.

Status is merged across sub-tables using a severity-wins rule:

```abap
ev_status = COND #( WHEN ev_status = 'E' OR lv_st = 'E' THEN 'E'
                    WHEN ev_status = 'A' OR lv_st = 'A' THEN 'A'
                    ELSE lv_st ).
```

Messages from all matching sub-tables are accumulated with `APPEND LINES OF ... TO et_msgs`.

## Final state of create_csv_err_body

```abap
METHOD create_csv_err_body.
  DATA: lt_msgs   TYPE bapiret2_tt,
        lv_status TYPE /aif/proc_status,
        lv_errtxt TYPE string.

  rv_has_errors = abap_false.

  LOOP AT it_lst ASSIGNING FIELD-SYMBOL(<ls_lst>).
    get_err_for_glblid(
      EXPORTING
        is_data   = is_data
        iv_glblid = <ls_lst>-glblid
      IMPORTING
        ev_status = lv_status
        et_msgs   = lt_msgs ).

    CHECK lv_status = 'E' OR lv_status = 'A'
       OR ( lv_status IS INITIAL
            AND ( line_exists( lt_msgs[ type = 'E' ] )
                  OR line_exists( lt_msgs[ type = 'A' ] ) ) ).
    rv_has_errors = abap_true.

    DATA(ls_err) = COND bapiret2(
      WHEN line_exists( lt_msgs[ type = 'E' ] ) THEN lt_msgs[ type = 'E' ]
      WHEN line_exists( lt_msgs[ type = 'A' ] ) THEN lt_msgs[ type = 'A' ] ).

    DATA(lv_errnr) = COND string(
      WHEN ls_err IS NOT INITIAL THEN |{ ls_err-id }/{ ls_err-number }| ).
    CLEAR lv_errtxt.
    IF ls_err IS NOT INITIAL.
      MESSAGE ID ls_err-id TYPE ls_err-type
        NUMBER ls_err-number
        WITH ls_err-message_v1 ls_err-message_v2
             ls_err-message_v3 ls_err-message_v4
        INTO lv_errtxt.
    ENDIF.

    APPEND escape_csv( |{ <ls_lst>-typ }|     ) && ';' &&
           escape_csv( |{ <ls_lst>-quelle }|  ) && ';' &&
           escape_csv( |{ <ls_lst>-satznr }|  ) && ';' &&
           escape_csv( |{ <ls_lst>-pos }|     ) && ';' &&
           escape_csv( |{ <ls_lst>-kap }|     ) && ';' &&
           escape_csv( |{ <ls_lst>-titel }|   ) && ';' &&
           escape_csv( |{ <ls_lst>-ukto }|    ) && ';' &&
           escape_csv( |{ <ls_lst>-oeh }|     ) && ';' &&
           escape_csv( |{ <ls_lst>-faellig }| ) && ';' &&
           escape_csv( |{ <ls_lst>-soll }|    ) && ';' &&
           escape_csv( lv_errnr               ) && ';' &&
           escape_csv( lv_errtxt              )
      TO ct_csv_table.
  ENDLOOP.
ENDMETHOD.
```

## Final state of GET_ERR_FOR_GLBLID

```abap
METHOD get_err_for_glblid.
  DATA lv_st TYPE /aif/proc_status.
  CLEAR: ev_status, et_msgs.

  IF line_exists( is_data-ao[ glblid = iv_glblid ] ).
    lv_st = is_data-ao[ glblid = iv_glblid ]-ao_proc_status.
    ev_status = COND #( WHEN ev_status = 'E' OR lv_st = 'E' THEN 'E'
                        WHEN ev_status = 'A' OR lv_st = 'A' THEN 'A'
                        ELSE lv_st ).
    APPEND LINES OF is_data-ao[ glblid = iv_glblid ]-msg TO et_msgs.
  ENDIF.

  IF line_exists( is_data-ao_reference[ glblid = iv_glblid ] ).
    lv_st = is_data-ao_reference[ glblid = iv_glblid ]-ao_proc_status.
    ev_status = COND #( WHEN ev_status = 'E' OR lv_st = 'E' THEN 'E'
                        WHEN ev_status = 'A' OR lv_st = 'A' THEN 'A'
                        ELSE lv_st ).
    APPEND LINES OF is_data-ao_reference[ glblid = iv_glblid ]-msg TO et_msgs.
  ENDIF.

  IF line_exists( is_data-mb[ glblid = iv_glblid ] ).
    lv_st = is_data-mb[ glblid = iv_glblid ]-mv_proc_status.
    ev_status = COND #( WHEN ev_status = 'E' OR lv_st = 'E' THEN 'E'
                        WHEN ev_status = 'A' OR lv_st = 'A' THEN 'A'
                        ELSE lv_st ).
    APPEND LINES OF is_data-mb[ glblid = iv_glblid ]-msg TO et_msgs.

    lv_st = is_data-mb_up[ glblid = iv_glblid ]-mv_up_proc_status.

  ENDIF.
```
